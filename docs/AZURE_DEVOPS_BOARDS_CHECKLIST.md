# Azure DevOps Boards — rubric backup checklist

Complete this in the **Azure DevOps** web UI (not in Git). Screen captures need your **Centennial ID** visible.

## Required (course rubric)

| Requirement | Your notes |
|-------------|------------|
| At least **4 work item types** (e.g. Epic, Feature, Bug, Task) | |
| **2 work items per type** (8 items minimum) | |
| **Kanban** board + **backlog** visible | |
| Each item **assigned** to a project member | |
| Each item moves through **all board stages** (e.g. New → Active → Resolved → Done — match your process) | |

## Suggested mapping to GitHub

Align Azure work items with your **GitHub Issues** so the demo tells one story (inventory features, bugs, Jenkins work).

## Partner split (example)

- Person A: creates/types Epic + Features, moves items on the board during sprint.
- Person B: Bugs + Tasks, ensures assignments and “Done” states for demo.
